# SideNote

SideNoteは、[Obsidian](https://obsidian.md)のプラグインで、ノートにコメントを追加することができます。これらのコメントは専用のサイドペインに表示され、メインのテキストを乱雑にすることなく、注釈を簡単に確認・ナビゲートできます。コメントはエディタ内で直接ハイライト表示されるため、視覚的に素早く参照できます。

## 主な機能

### 基本機能

- **選択範囲へのコメント追加**: Markdownノート内の選択したテキストに簡単にコメントを追加できます
- **ノートコメント**: テキストを選択せずにノート全体へコメントを追加できます — エディタで何も選択しない状態で右クリック、またはコマンドパレットから「Add note comment」を実行
- **2つの表示モード**:
  - **サイドバーモード**: 右サイドバーにコメントを開いて常時表示
  - **スプリットビューモード**: ノートの横に分割ペインとしてコメントを開く
- **視覚的なハイライト**: コメントしたテキストがエディタ内で自動的にハイライト表示（黄色の背景と下線）
- **自動追跡**: ノートを編集してもハッシュベースのマッチングによりコメントが自動的にテキストを追跡
- **クリックで移動**:
  - サイドペインの任意のコメントをクリックすると、エディタ内の該当箇所にジャンプ
  - コメントを**ダブルクリック**すると直接編集モーダルが開く
  - エディタ内のハイライトされたテキストをクリックすると、サイドバーが開き対応するコメントがハイライトされます
- **内部リンク対応**: コメント本文内の `[[WikiLink]]` がクリック可能になり、リンク先のノートを開ける
- **検索フィルター**: サイドバー上部の検索欄にキーワードを入力するとコメントをリアルタイム絞り込み（ハイライトテキストとコメント本文が対象。Current File・All Notes 両モードで動作）
- **編集と削除**: サイドペインから直接コメントを管理できます
- **インラインエディット**: コメントの追加・編集はサイドバー内のインラインテキストエリアで直接行います — モーダル不要。`Cmd/Ctrl + Enter` で保存、`Esc` でキャンセル
- **柔軟なソート**: コメントをファイル内の位置順または作成日時順で並べ替え可能
- **オーファン化されたコメントの管理**: 元のテキストが削除されると、コメントは「オーファン化」され、個別に管理できます

### 高度な機能

- **Operational Transformation 追跡**: コメントの位置は絶対文字オフセットとして保存され、CodeMirror 6 の `ChangeSet.mapPos()` を使ってあらゆるドキュメント編集に追従します。
- **正確な重複テキスト対応**: 同じ文字列が複数箇所に存在しても、各コメントは自分のテキストインスタンスを追跡します。ハイライトが別の出現箇所に移動することはありません。
- **即時オーファン検出**: コメント対象テキストが削除されると範囲が直ちに潰れ（collapse）、別の出現箇所に移ることなく赤いマーカーが表示されます。赤いドットはその後の編集にも OT で追従し続けます。
- **Undo でのリアルタイム復帰**: Ctrl+Z で削除したテキストが復元されると、ファイル保存を待たずにリアルタイムでコメントが黄色に戻ります。
- **セッションをまたいだオフセット保存**: `startOffset`/`endOffset`（絶対文字オフセット）をコメントごとに保存するため、Obsidian 再起動後も再検索なしで正確な位置を復元できます。
- **アクティブファイルの自動更新**: サイドバーモード使用時、ファイルを切り替えるとコメントビューが自動更新
- **オーファン化されたコメントのハイライト**: 削除されたテキストの位置に赤い文字1文字でマーク（設定でオフ可能）
- **オプションのMarkdown保存**: コメントをノートごとのサイドノートMarkdownファイルに保存可能（デフォルトは `side-note-comments` フォルダ）

## 使い方

### 選択したテキストへのコメント追加

1. エディタで**テキストを選択**（推奨は3文字以上、正確性のためには10文字以上）
2. 選択したテキストを**右クリック**し、「Add comment to selection」を選択
   - またはコマンドパレット（`Cmd/Ctrl + P`）→「Side Note: Add comment to selection」
3. サイドバーにインラインフォームが表示されるので、コメントを入力
   - `Cmd/Ctrl + Enter` で保存
   - `Esc` でキャンセル
4. テキストが自動的に黄色の背景と下線でハイライトされます

### ノートコメントの追加（テキスト選択不要）

1. エディタの任意の場所をクリック（テキストは選択しない）
2. **右クリック**して「Add note comment」を選択
   - またはコマンドパレット → 「Side Note: Add note comment」
3. サイドバーに表示されるインラインフォームにコメントを入力
4. `Cmd/Ctrl + Enter` で保存、`Esc` でキャンセル

ノートコメントはファイル全体に紐付けられ、特定の文章には紐付きません。サイドバーではファイル名をタイトルとして表示され、青い左ボーダーが付きます。クリックするとファイルの先頭に移動します。

### コメントの表示

**オプション1: サイドバーモード**
- リボンのメッセージスクエアアイコンをクリック
- またはコマンドパレットから「Side Note: Open in Sidebar」を実行
- ビューはサイドバーに表示され、ファイルを切り替えると自動的に更新されます

**オプション2: スプリットビューモード**
- コマンドパレットから「Side Note: Open in Split View」を実行
- 現在のファイルのコメントを表示する新しいペインが右側に開きます

### コメント間の移動

- サイドペインの任意のコメントをクリックすると、エディタ内の該当箇所にジャンプ
- コメントを**ダブルクリック**すると直接編集モーダルが開く
- エディタ内のハイライトされたテキストをクリックすると、サイドバーが開き（まだ開いていない場合）、対応するコメントがハイライトされます
- コメントはテキスト内で直接ハイライトされているため、視覚的に簡単に参照できます

### コメントの検索

Side Note パネル上部の検索欄にキーワードを入力すると、コメントをリアルタイムで絞り込めます。ハイライトされたテキストとコメント本文の両方が検索対象です。Current File・All Notes の両モードで動作します。日本語・中国語・韓国語などの IME 入力にも完全対応しており、変換中に絞り込みが割り込むことはありません。

### コメントの管理

- **編集**: `...` メニュー → Edit、またはコメントを**ダブルクリック** — サイドバーにインラインテキストエリアが表示されます。`Cmd/Ctrl + Enter` で保存、`Esc` でキャンセル
- **削除**: 各コメントの `...` メニュー → Delete
- **ソート**: 設定 → Comment sort order で並び順を変更（位置順またはタイムスタンプ順）

### 設定

設定 → Side Note からアクセス：

- **Comment Sort Order**: ファイル内の位置順またはタイムスタンプ順を選択
- **Show Highlights in Editor**: エディタ内の視覚的なハイライトのオン/オフ切り替え
- **Store comments as Markdown files**: コメントをノートごとのサイドノートMarkdownファイルに保存
- **Markdown comments folder**: サイドノートMarkdownファイルを保存するフォルダ（Vault相対）を指定
- **Orphaned Comments**: オーファン化されたコメントの数を表示し、一括削除可能

## モバイル対応

**SideNote v1.0.3以降**は、iOS（Obsidian モバイルアプリ）とAndroidデバイスの両方に完全対応しました！

### モバイル機能

- **レスポンシブデザイン**: コメントモーダルはモバイル画面サイズに自動的に適応
- **タッチフレンドリーなボタン**: すべてのボタンに適切なタッチターゲットサイズ（最小44px）があり、簡単にタップ可能
- **モバイルキーボード最適化**: テキスト入力はモバイルキーボード向けに最適化され、iOSの自動ズーム防止のため16pxフォントサイズを使用
- **改善されたフォーカス管理**: タッチデバイスでのシームレスなモーダル操作のための、より良いフォーカス処理
- **テキスト選択サポート**: モバイルエディタでのテキスト選択とコメント追加に完全対応

### モバイルでコメントを追加する方法

1. ノートを編集モードで開く
2. コメントを付けたい**テキストを長押し**して選択
   - テキストの開始位置で長押しし、終了位置までドラッグします
3. **エディタ上部のツールバー**に表示される**メッセージアイコン（💬）**をタップ
   - ツールバーにアイコンが表示されない場合は、ツールバーを左右にスワイプしてください
4. コメント入力モーダルが表示されるので、コメントを入力
5. **Add**ボタンをタップして保存

### モバイルでコメント一覧を開く方法

#### **サイドバーでコメント一覧を表示**

**iOS（Obsidian iOS版）:**
1. 画面左上のハンバーガーメニュー（≡）をタップ
2. 右にスワイプするか、「Open in Sidebar」コマンドを実行
3. サイドバーにコメント一覧が表示される

**Android（Obsidian Android版）:**
1. 画面上部のメニュー（≡）をタップ
2. 「Side Note: Open in Sidebar」コマンドを実行（検索して実行）
3. 右側のパネルにコメント一覧が表示される

#### **コマンドパレットから直接開く**

1. 画面上部のメニュー（≡ または ⋮）をタップ
2. コマンドパレットを開く
3. 「side note」で検索
4. 「Side Note: Open in Sidebar」をタップ
5. サイドバーが開き、現在のノートのコメント一覧が表示される

### コメント一覧からコメントをクリック

- サイドバーで各コメントをタップすると、エディタ内でそのコメント位置にジャンプ
- コメント横の**編集（鉛筆）**アイコンをタップするとコメントを編集
- コメント横の**削除（ゴミ箱）**アイコンをタップするとコメントを削除

### モバイルのトラブルシューティング

- **ツールバーにメッセージアイコンが表示されない**: ツールバーを左右にスワイプして探してください。Obsidianのモバイル設定でツールバーのカスタマイズが可能です。
- **「Add」ボタンが反応しない**: ボタンをしっかりタップしてください。コメント欄にテキストが入力されていることを確認してください（空のコメントは保存できません）。
- **テキスト選択が困難**: より長いテキスト（10文字以上）を選択してください。より確実なマッチングが期待できます。
- **サイドバーが表示されない**: 画面上部のメニュー（≡）からコマンドパレットを開き、「Side Note: Open in Sidebar」を実行してください。

## 重要な注意事項

### テキスト追跡の制限事項

**短いテキスト（10文字未満）**: 非常に短いテキストへのコメントは動作が不安定になったり、近くの同一テキストに飛ぶことがあります。最良の結果を得るには、コメント時に少なくとも10文字以上を選択してください。

**重複テキスト**: OT ベースの追跡により、編集セッション中は各コメントが自分のテキストインスタンスに追従し続けます。Obsidian 再起動後やファイルを外部で編集した場合は、まず保存済みの絶対オフセットを検証し、無効な場合のみ ±50 行の近接検索で位置を再アンカーします。元の位置から 50 行以内に同一テキストが存在すると稀に誤ったインスタンスにマッチすることがあります。

### 今後の計画

**CRDT への移行**: 現在の OT 実装（CM6 の `ChangeSet.mapPos()` を利用）は単一デバイスでの編集セッションにおいて良好に動作しますが、デバイス間の同期（例: Obsidian Sync）ではフォールバックの再検索に依存しています。将来的には、アノテーション追跡を完全な **CRDT（Conflict-free Replicated Data Type）** モデルへ移行することを計画しています。実装には VS Code Live Share などにも採用されている [Yjs](https://github.com/yjs/yjs) ライブラリを活用する方向で検討しています。CRDT 化により以下が実現されます。

- 位置情報を CRDT ドキュメントとして符号化することで再検索を完全に不要にし、複数デバイスでの同時オフライン編集後でも正確に収束する
- 同じノートを複数デバイスで同時編集してもコンフリクトなしに真のマルチデバイスアノテーション同期を実現する
- 再検索ウィンドウ内に同一テキストが存在する場合にコメントが誤った出現箇所に再アンカーされる残存エッジケースを解消する

**ハイライトのバリエーション**: カスタマイズ可能なハイライトの色とスタイルを追加し、異なるコメントタイプに対応する予定です。

**リッチな編集UI**: 新しいMarkdown保存オプションの上で、ショートカットやプレビューなど、よりリッチなMarkdown編集体験を追加する予定です。

## 技術的詳細

- コメントは選択テキスト・SHA256 ハッシュ・絶対文字オフセット（`startOffset`/`endOffset`）・行/文字座標とともに `data.json` に保存されます
- **ハイライト追跡**は Operational Transformation（OT）で実現: 絶対オフセットをドキュメントへの変更のたびに CodeMirror 6 の `ChangeSet.mapPos()` で変換 — 編集セッション中にテキスト検索は行いません
- `mapPos` のバイアスは範囲開始に `assoc=+1`、範囲終了に `assoc=-1` を使用し、ハイライト境界への挿入がトラッキング範囲外に留まることを保証します
- オーファン化されたコメント（赤いマーカー）も OT トラッカーに残り続け、その後の編集に追従。元のテキストが再出現すると（例: Ctrl+Z）即座に黄色に復帰します
- コールドスタートや外部ファイル変更時は保存済みの絶対オフセットを先に検証し、無効な場合のみ ±50 行近接 SHA256 検索を実行します
- エディタ内のハイライトには CodeMirror 6 のデコレーション機能を使用

## バージョン履歴

### 1.0.8
- **サイドバーのコメントクリック時のスクロール位置を修正** — コメントをクリックすると、ハイライト箇所がエディタの中央に表示されるように
  - `scrollIntoView` の直後に `editor.focus()` を呼んでいたため、ブラウザのフォーカス起因スクロール（nearest モード）がセンタリングを上書きしていた
  - `setActiveLeaf` が既にフォーカスを処理しているため `focus()` を削除し、`setActiveLeaf` の副作用が完了してからスクロールを実行するよう短い遅延を追加
- **コメント本文内の `![[embed]]` 埋め込み展開に対応**
  - `![[note.md]]` がコメントカード内にインライン表示されるようになった（従来はプレーンなwikiリンクに変換していた）
  - `MarkdownRenderer.render` に正しい `sourcePath` と `component` を渡すことで埋め込みコンテンツの解決・表示が可能になった
  - 非推奨の `MarkdownRenderer.renderMarkdown` から現行の `MarkdownRenderer.render` API に移行
- **コメント本文内のコールアウトと引用のスタイルを修正**
  - `> [!NOTE]` などのコールアウトや `> ...` の引用ブロックがスタイルなしのプレーンテキストで表示されていた問題を修正
  - 原因：ObsidianのCSSはコールアウト・引用のスタイルを `.markdown-rendered` という祖先クラスにスコープしており、コメントのコンテンツラッパーにそのクラスが付いていなかった
  - コンテンツラッパーに `markdown-rendered` を追加し、余分なマージンでコメントカードが膨らまないようCSSも調整

  - `[[#heading]]` リンクが同一ファイル内の見出しへ移動するように
- **コメント追加・編集をポップアップモーダルからインラインサイドバー編集に変更**
  - コメントの追加と編集がサイドバー内のインラインテキストエリアで直接行えるようになり、モーダルは表示されない
  - ノートコメント・テキスト選択コメントのどちらでも動作
  - `Ctrl/Cmd+Enter` で保存（グローバルホットキーとの競合を避けるため Obsidian の `Scope` システムを経由）、`Esc` でキャンセル
  - バックグラウンドイベント（ファイル保存など）でサイドバーが再描画されても入力中のドラフトが保持される
- **コメント内の外部URLおよびObsidianプロトコルリンクが正しく開かない問題を修正**（issue [#25](https://github.com/mofukuru/SideNote/issues)）
  - `https://`・`http://`・`obsidian://` などで始まるリンクがエラーになる代わりに、システムブラウザ/アプリで正しく開くように修正
  - 従来はすべての href を内部リンク専用の `openLinkText` に渡していたことが原因
- **コメント内のwikiリンクをCtrl/Cmd+Clickで新しいタブに開く機能を追加**（issue [#24](https://github.com/mofukuru/SideNote/issues)）
  - Ctrl（Windows/Linux）または Cmd（Mac）を押しながらコメント内の `[[WikiLink]]` をクリックすると、現在のタブを置き換えず新しいタブでノートを開きます
- **OT mapPos バイアスのバグを修正 — ハイライト境界での編集でオーファン化しなくなった**（issue [#26](https://github.com/mofukuru/SideNote/issues)）
  - `ChangeSet.mapPos()` の `assoc` 引数が逆になっており、`from` に `-1`（左バイアス）、`to` に `+1`（右バイアス）を使用していた
  - これによりハイライトの開始または終了位置への挿入が追跡範囲を拡大させ、テキスト検証が失敗して即座にオーファン化する問題が発生していた
  - `from` を `assoc=+1`、`to` を `assoc=-1` に修正し、境界への挿入が範囲外に留まるよう修正
- **オーファン化された（赤い）ハイライトも OT で追従するように改善**（issue [#26](https://github.com/mofukuru/SideNote/issues)）
  - 従来、赤いドットはオーファン化した瞬間の位置に凍結され、その後の編集でもずれたまま表示されていた
  - 新しい `orphanedAt` トラッカーがオーファン位置に対しても `ChangeSet.mapPos()` を適用し続けるため、赤いドットが正しい位置を追う
  - Undo 回復はリアルタイムで動作: 追跡位置に元のテキストが再出現するとファイル保存を待たずに即座に黄色に戻る
- **セッション横断的な精度向上のため絶対オフセットを保存**（issue [#26](https://github.com/mofukuru/SideNote/issues)）
  - 新しい `startOffset`/`endOffset` フィールドを `data.json` に保存し、OT トラッカーが変更ごとに更新
  - Obsidian 再起動時は保存済みオフセットを先に検証し、古い場合のみ再検索を実行
- **外部同期の再検索ウィンドウを ±10 行から ±50 行に拡大**
  - 他のデバイスでノートを編集してコメント対象テキストが大きく移動した場合の復元精度が向上
